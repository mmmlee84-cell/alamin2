# 간편 알람 웹앱 기술명세서

> 경제 뉴스 자동 프리핑(Prefetch) 기능 추가 명세

- 문서 버전: v1.0
- 작성일: 2026-07-24
- 대상 서비스: 간편 알람 (Simple Alarm) 웹앱

---

## 1. 개요

기존 간편 알람 웹앱은 알람 등록/관리와 서울 현재 기온 표시 기능을 제공한다.
본 명세서는 여기에 **최근 경제 뉴스 5개를 자동으로 불러와 표시하는 기능**을 추가하기 위한 기술 사양을 정의한다.

### 1.1 기존 기능

| 구분 | 내용 |
|------|------|
| 알람 등록 | 시간 입력, 라벨(선택), 요일(일~토) 반복 설정 후 등록 |
| 알람 목록 | 등록된 알람 표시 (없을 경우 "등록된 알람이 없습니다." 표기) |
| 현재 시각 | 실시간 시계 표시 (예: 15:05:43) |
| 날씨 | 서울 현재 기온 표시 (예: 31.4℃) |

### 1.2 추가 기능

- 최근 경제 뉴스 **5개**를 페이지 진입 시 자동 로딩(프리핑)
- 일정 주기(기본 10분)마다 자동 갱신
- 뉴스 제목 클릭 시 원문 기사로 이동(새 탭)

## 2. 왜 뉴스는 날씨처럼 간단하지 않은가

기온 표시는 Open-Meteo 등 **CORS를 허용하는 무료 API**를 브라우저에서 직접 `fetch`로 호출하는 구조로 추정된다.
반면 대부분의 뉴스 소스(언론사 RSS, 네이버 뉴스 API 등)는 브라우저에서 직접 호출 시 **CORS 정책에 의해 차단**된다.

따라서 다음 세 가지 방식 중 하나를 선택해야 한다.

### 2.1 방식 비교

| 항목 | 방법 A: 프록시 서비스 | 방법 B: 서버리스 백엔드 (RSS) | 방법 C: 서버리스 백엔드 (네이버 API) |
|------|----------------------|------------------------|------------------------|
| 구조 | 브라우저에서 프록시(rss2json 등) 경유 호출 | Vercel/Netlify/Cloudflare Workers 함수 경유 | Vercel/Netlify/Cloudflare Workers 함수 경유 |
| 백엔드 필요 | 불필요 (현재 구조 유지) | 필요 (함수 1개) | 필요 (함수 1개) |
| CORS 처리 | 프록시가 해결 | 함수에서 헤더 설정 | 함수에서 헤더 설정 |
| API 키 은닉 | 불가 (키 불필요) | 가능 | **필수** (Client ID/Secret 서버 보관) |
| 안정성 | 무료 프록시 제한(속도/호출량) 존재 | 안정적 | 안정적 (일 호출 한도 존재) |
| 키워드 검색 | 불가 (RSS 고정) | 불가 (RSS 고정) | 가능 |
| 권장 용도 | 프로토타입 / 개인용 | 실서비스 / 배포용 | 실서비스 / 키워드 검색 필요 시 |

> **결론:** 초기 구현 및 현재 구조 유지 목적이면 **방법 A**, 실제 배포 및 안정성이 필요하면 **방법 B**, 키워드 기반 검색과 네이버 API 연동이 필요하면 **방법 C**를 채택한다. 상세 사양은 3장(방법 A), 4장(방법 B), 5장(방법 C)을 참고한다.

## 3. 방법 A — 프록시 서비스 (권장: 초기 구현)

기존 HTML/JS 파일에 코드만 추가하면 되며, 별도 서버가 필요 없다.

### 3.1 UI 마크업

기온 박스 아래에 다음 HTML을 추가한다.

```html
<div id="news-box" style="max-width:410px;margin:16px auto;background:#2b3245;
     border-radius:12px;padding:16px 20px;color:#cbd3e1;">
  <div style="font-weight:700;color:#7ec8f5;margin-bottom:10px;">📈 최근 경제 뉴스</div>
  <div id="news-list">불러오는 중…</div>
</div>
```

### 3.2 스크립트

```javascript
// 경제 RSS (원하는 언론사로 교체 가능)
const NEWS_RSS = 'https://www.hankyung.com/feed/economy'; // 한국경제 경제면
// 예: 매일경제 'https://www.mk.co.kr/rss/30100041/'

async function loadEconomyNews() {
  const box = document.getElementById('news-list');
  try {
    const url = 'https://api.rss2json.com/v1/api.json?rss_url='
                + encodeURIComponent(NEWS_RSS);
    const res = await fetch(url);
    const data = await res.json();
    const items = (data.items || []).slice(0, 5); // 최근 5개

    if (!items.length) { box.textContent = '뉴스를 가져오지 못했습니다.'; return; }

    box.innerHTML = items.map(it => `
      <a href="${it.link}" target="_blank" rel="noopener"
         style="display:block;padding:6px 0;color:#cbd3e1;text-decoration:none;
                border-bottom:1px solid #3a4257;font-size:14px;">
        ${it.title}
      </a>`).join('');
  } catch (e) {
    box.textContent = '뉴스를 불러오지 못했습니다.';
  }
}

loadEconomyNews();                    // 최초 실행 (프리핑)
setInterval(loadEconomyNews, 600000); // 10분마다 자동 갱신
```

### 3.3 동작

1. 페이지 로딩 시 `loadEconomyNews()`가 즉시 실행되어 뉴스 5개를 프리핑한다.
2. `setInterval`로 10분(600,000ms)마다 자동 재호출하여 목록을 갱신한다.
3. 언론사 변경은 `NEWS_RSS` 값만 교체하면 된다.

## 4. 방법 B — 서버리스 백엔드 (권장: 실서비스)

Vercel / Netlify / Cloudflare Workers에 함수 하나를 배포하여 뉴스를 받아 프론트로 전달한다.

### 4.1 처리 흐름

```
[브라우저]  →  [서버리스 함수]  →  [언론사 RSS / 뉴스 API]
     ↑                                      │
     └──────────  JSON 응답 (CORS 허용) ─────┘
```

### 4.2 서버리스 함수 예시 (Vercel, /api/news.js)

```javascript
// /api/news.js
export default async function handler(req, res) {
  try {
    const RSS = 'https://www.hankyung.com/feed/economy';
    const r = await fetch('https://api.rss2json.com/v1/api.json?rss_url='
                          + encodeURIComponent(RSS));
    const data = await r.json();
    const items = (data.items || []).slice(0, 5).map(it => ({
      title: it.title,
      link: it.link,
    }));

    res.setHeader('Access-Control-Allow-Origin', '*');
    res.setHeader('Cache-Control', 's-maxage=600'); // 10분 캐시
    res.status(200).json({ items });
  } catch (e) {
    res.status(500).json({ items: [], error: '뉴스 로딩 실패' });
  }
}
```

### 4.3 프론트 호출부

```javascript
async function loadEconomyNews() {
  const box = document.getElementById('news-list');
  try {
    const res = await fetch('/api/news');
    const { items } = await res.json();
    if (!items.length) { box.textContent = '뉴스를 가져오지 못했습니다.'; return; }
    box.innerHTML = items.map(it => `
      <a href="${it.link}" target="_blank" rel="noopener"
         style="display:block;padding:6px 0;color:#cbd3e1;text-decoration:none;
                border-bottom:1px solid #3a4257;font-size:14px;">
        ${it.title}
      </a>`).join('');
  } catch (e) {
    box.textContent = '뉴스를 불러오지 못했습니다.';
  }
}
loadEconomyNews();
setInterval(loadEconomyNews, 600000);
```

## 5. 방법 C — 네이버 뉴스 검색 API

네이버 개발자센터에서 애플리케이션을 등록하면 뉴스 검색 API를 사용할 수 있다. RSS 프록시보다 검색 정확도가 높고 키워드 기반 검색이 가능하다는 장점이 있으나, **Client ID / Client Secret 인증이 필수**이므로 반드시 서버(방법 B와 동일한 서버리스 백엔드) 경유로만 호출해야 한다.

### 5.1 API 개요

| 항목 | 내용 |
|------|------|
| 엔드포인트 | `GET https://openapi.naver.com/v1/search/news.json` |
| 필수 헤더 | `X-Naver-Client-Id`, `X-Naver-Client-Secret` |
| 주요 파라미터 | `query`(검색어), `display`(출력 건수, 최대 100), `sort`(`sim` 관련도순 / `date` 최신순) |
| 무료 호출 한도 | 애플리케이션당 일 25,000회 (네이버 정책에 따라 변경 가능) |

### 5.2 서버리스 함수 예시 (Vercel, /api/news.js)

```javascript
// /api/news.js
export default async function handler(req, res) {
  try {
    const query = encodeURIComponent('경제');
    const r = await fetch(
      `https://openapi.naver.com/v1/search/news.json?query=${query}&display=5&sort=date`,
      {
        headers: {
          'X-Naver-Client-Id': process.env.NAVER_CLIENT_ID,
          'X-Naver-Client-Secret': process.env.NAVER_CLIENT_SECRET,
        },
      }
    );
    const data = await r.json();
    const items = (data.items || []).slice(0, 5).map(it => ({
      title: it.title.replace(/<[^>]+>/g, ''), // HTML 태그 제거
      link: it.originallink || it.link,
    }));

    res.setHeader('Access-Control-Allow-Origin', '*');
    res.setHeader('Cache-Control', 's-maxage=600');
    res.status(200).json({ items });
  } catch (e) {
    res.status(500).json({ items: [], error: '뉴스 로딩 실패' });
  }
}
```

### 5.3 주의사항

- **`NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`는 서버리스 함수의 환경 변수로만 저장하며, 프론트엔드 코드(HTML/JS/브라우저 네트워크 요청)에 절대 포함되어서는 안 된다.**
- 프론트엔드는 방법 B의 `/api/news` 호출부(4.3절)를 그대로 재사용할 수 있다.
- 방법 A(무료 RSS 프록시)와 달리 키 발급/등록 절차가 필요하므로, 초기 프로토타입 단계보다는 정식 서비스 단계에서 채택을 권장한다.

## 8. 제약 및 주의사항

### 8.1 기술적 제약

- **무료 프록시 제한:** rss2json 등 무료 프록시는 호출 횟수·속도 제한이 있다. 이용자 증가 시 방법 B로 전환 권장.
- **RSS 안정성:** 언론사가 RSS 포맷/주소를 변경하면 로딩 실패 가능. 예외 처리(`try/catch`)로 대응.
- **네이버 API 호출 한도:** 방법 C 채택 시 애플리케이션당 일 25,000회 한도(정책 변경 가능)를 초과하지 않도록 캐싱/호출 주기를 관리해야 한다.

### 8.2 저작권 유의사항

- 언론사 기사 본문을 그대로 재게시하는 것은 **저작권 침해 소지**가 있다.
- 본 명세는 **기사 제목 + 원문 링크**만 노출하고, 클릭 시 원 기사로 이동시키는 방식을 채택하여 리스크를 최소화한다.
- 각 언론사의 RSS 이용 약관을 사전 확인할 것. 방법 C는 네이버 오픈 API 이용약관을 사전 확인할 것.

## 9. 알람 연동 브리핑

알람이 울리는 시점에 경제 뉴스 브리핑을 함께 제공하는 기능에 대한 사양이다.

### 9.1 개요

알람 발생 시, 별도 조작 없이 최근 경제 뉴스 5개(2장~5장에서 정의한 방법 A/B/C 중 하나로 조회)를 알람 화면에 함께 노출한다.

### 9.2 트리거 조건

- 알람이 설정된 시각에 도달하여 알람 소리 재생이 시작되는 시점을 트리거로 한다.

### 9.3 표시 방식

- 알람 화면 내 뉴스 박스(`#news-box`)에 텍스트 목록 형태로 노출한다.
- 음성 낭독(TTS)은 본 버전 범위에 포함하지 않는다.

### 9.4 알람 소리와의 관계 (원칙)

- 뉴스 브리핑은 알람 소리 재생과 **독립적인 프로세스**로 동작한다.
- 뉴스 브리핑 표시/로딩 실패가 알람 소리 재생을 지연시키거나 중단시켜서는 안 된다.

### 9.5 실패 처리

- 뉴스 API 응답 지연 또는 실패 시, 뉴스 영역에는 안내 문구("뉴스를 불러오지 못했습니다.")만 표시하고 알람 재생 로직에는 어떠한 영향도 주지 않는다.

### 9.6 재생 타이밍 (미확정)

다음 사항은 아직 결정되지 않았으며, 별도 논의가 필요하다.

- 알람 소리와 뉴스 브리핑 표시가 **동시에** 시작되는지, 아니면 알람을 정지한 **이후**에 브리핑이 나타나는지 순서 미정.
- 스누즈(다시 알림) 시 뉴스 브리핑을 매번 다시 갱신할지, 최초 1회만 표시할지 미정.
- 향후 음성 안내(TTS) 도입 시, 알람 소리와 음성 브리핑이 겹치지 않도록 어떤 순서/딜레이로 재생할지 미정.

## 10. 뉴스 검색 키워드 (미확정)

방법 C(네이버 뉴스 검색 API) 채택 시 사용할 검색 키워드/정렬 기준은 아직 확정되지 않았다.

- 고정 키워드("경제" 등 단일 키워드) vs 다중 키워드(예: "경제", "증시", "부동산" 조합) 여부 미정.
- 정렬 기준을 관련도순(`sim`)으로 할지 최신순(`date`)으로 할지 미정.
- 사용자가 키워드를 직접 설정하도록 할지, 앱에서 고정값으로 제공할지 미정.
- 카테고리 선택 토글(11장 참고) 도입 여부에 따라 키워드 체계가 달라질 수 있음.

## 11. 향후 개선 아이디어

- 뉴스 카테고리 선택(경제/증시/부동산 등) 토글 추가
- 로딩 실패 시 재시도(backoff) 로직
- 여러 언론사 소스를 병합하여 최신순 정렬
