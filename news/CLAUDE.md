# news/CLAUDE.md

이 문서는 `news/` 폴더 안의 파일을 다룰 때만 적용되는 세부 규칙이다.
보안 등 프로젝트 전체 공통 규칙은 루트 `CLAUDE.md`를 따르며 여기서는 반복하지 않는다.

## 담당 범위

이 폴더는 알람 앱의 **'최근뉴스' 조회 및 TTS 브리핑 기능**을 담당한다. 그 외 알람 등록/관리, 시각/기온 표시 등은 이 폴더의 책임이 아니다.

## 파일 구조 규칙

- 이 폴더에는 `news.js`, `news.css` 파일만 둔다.
- 다른 파일(예: `prd.md`, 다른 CLAUDE.md 등 문서 파일 제외)을 추가하지 않는다.

## 함수 이름 규칙

- 데이터를 가져오는 함수: `fetchXxx` (예: `fetchNews`)
- 화면에 그리는 함수: `renderXxx` (예: `renderNewsList`)
- 순수 변환 함수(부수효과 없이 입력을 가공해 반환): `buildXxx` / `stripXxx` (예: `buildNewsItem`, `stripHtmlTags`)
- 위 세 범주 중 어디에도 속하지 않는 함수를 새로 만들 때는 임의로 접두어를 짓지 말고 먼저 사용자에게 확인한다.

## 함수 시그니처 규칙

- 함수 시그니처(파라미터/응답 필드명)는 기술명세서.md **5번(방법 C — 네이버 뉴스 검색 API)**과 **9.4번(알람 소리와의 관계 원칙)**을 그대로 따른다.
  - 데이터 필드명은 `title`, `link`(6.1절 데이터 모델과 동일)를 그대로 사용한다.
  - 네이버 API 관련 파라미터명은 `query`, `display`, `sort`, 응답 파싱 시 `items` 키를 임의로 바꾸지 않는다(5.1/5.2절 그대로).
  - 9.4절의 독립성 원칙에 따라, 뉴스 조회/렌더링 함수는 알람 재생 로직과 파라미터를 공유하거나 알람 재생 함수를 인자로 받지 않는다. 뉴스 함수는 알람 로직을 몰라도 독립적으로 호출 가능해야 한다.
- 위 두 절에 없는 시그니처가 필요하면 임의로 파라미터 이름을 정하지 말고 먼저 확인한다.

## script.js와의 관계

- `script.js`(알람 로직)는 이 폴더의 함수를 **호출만** 한다.
- 뉴스 조회, 렌더링, 데이터 가공 등 뉴스 관련 로직을 `script.js`에 직접 작성하지 않는다. 관련 로직은 반드시 `news.js`에 작성한다.
