---
name: github-upload
description: 사용자가 GitHub 저장소 주소(URL, 예: https://github.com/아이디/저장소이름.git)를 입력했을 때 사용하는 스킬. 프로젝트를 해당 주소로 안전하게 커밋·업로드(push)하는 절차를 담당한다. "여기에 올려줘", "깃허브에 업로드해줘"와 함께 저장소 주소가 주어지면 트리거된다.
---

절차:
1) git status로 변경 파일 확인 → .env, node_modules 등 민감/불필요
   파일 있으면 멈추고 알려줘
2) .gitignore에 .env 있는지 확인, 없으면 추가
3) git 초기화 안 됐으면 git init
4) 입력받은 주소를 origin으로 연결 (이미 있으면 같은지 확인 후 진행)
5) 커밋 메시지는 한국어로 간단히 요약 (사용자가 주면 그걸 우선)
6) 브랜치명(main/master) 확인 후 push
7) 에러 나면 쉬운 말로 원인과 해결법 설명

규칙:
- .env·API 키·비밀번호 포함된 커밋은 절대 금지
- force push는 명시적 요청 없으면 금지
- 저장소 주소 형식이 이상하면 진행 말고 다시 물어보기
